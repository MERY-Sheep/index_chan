# 日本語出力設定

ユーザは日本語で対応します。ユーザーが確認するものは日本語で書いてください。


## 出力ルール

- すべてのタスク実行結果、説明を日本語で出力してください
- 技術的な説明も日本語で行ってください

## 例外
- コード内のコメントは英語で構いません。
- 変数名や関数名は英語のままで構いません
- コード内の識別子（クラス名、関数名、変数名など）は英語のまま
- 説明やコメントは日本語で統一してください

## ドキュメント

- ドキュメントは日本語で統一してください
- コミットメッセージも日本語で記述してください


gitコマンドの中には、出力が長い場合に自動的にページャー（lessやmoreなどの対話的なビューアー）を起動するものがあります。特に以下のコマンドで問題が発生します：

git branch -a（ブランチが多い場合）
git log
git diff（変更が多い場合）
git show
これらのコマンドは対話的な入力を待つため、kiroの実行環境では応答が止まってしまいます。

回避策： 常に --no-pager オプションを使用することで、この問題を避けられます：

git --no-pager branch -a
git --no-pager log
git --no-pager diff
git --no-pager show

## 表示形式

設定値や構造化データを表示する際は、テーブル形式ではなく階層形式を使用してください。

```
設定名
├─ 項目1: 値1
├─ 項目2: 値2
└─ 項目3: 値3
```

テーブル（| 項目 | 値 |）は避けてください。
